# Matrix Multiplication: CPU vs GPU (CUDA)

В данной работе реализовано умножение матриц на CPU и GPU с использованием библиотеки Numba CUDA. Основная цель — сравнить производительность последовательной и параллельной реализации и проанализировать эффективность распараллеливания.

Умножение матриц — вычислительно затратная операция со сложностью O(N³). Каждый элемент результирующей матрицы вычисляется как сумма произведений элементов строки матрицы A и столбца матрицы B. При увеличении размера матриц время выполнения быстро возрастает, что делает задачу подходящей для параллельных вычислений.

Реализация на CPU выполнена классическим способом с использованием трёх вложенных циклов. Вычисления происходят последовательно: сначала выбирается строка, затем столбец, после чего выполняется суммирование произведений. Такая реализация проста, но плохо масштабируется при увеличении размера данных.

Реализация на GPU выполнена с использованием CUDA через библиотеку numba.cuda. Перед выполнением данные копируются на GPU, затем запускается CUDA-ядро, после чего выполняется синхронизация и копирование результата обратно на CPU. Время GPU измеряется только для выполнения ядра.

Ключевой особенностью GPU-реализации является распараллеливание. Оно происходит на уровне элементов результирующей матрицы: каждый поток GPU вычисляет один элемент C[row][col]. Координаты элемента определяются с помощью cuda.grid(2), что позволяет однозначно сопоставить поток и позицию в матрице.

Таким образом, вся матрица вычисляется одновременно большим количеством потоков. Потоки работают независимо, так как элементы результирующей матрицы не зависят друг от друга. Это делает задачу идеально подходящей для GPU.

Важно отметить, что внутри одного потока вычисления остаются последовательными: суммирование по индексу k не распараллелено. Следовательно, параллелизм реализуется на уровне элементов, а не внутри их вычисления.

В программе используется конфигурация CUDA с блоками размером 16×16 (256 потоков на блок). Размер сетки рассчитывается динамически, чтобы покрыть всю матрицу. Граничная проверка (row < rows и col < cols) необходима для предотвращения выхода за пределы массива.

Корректность результатов проверяется с помощью функции np.allclose(), что позволяет учитывать небольшую погрешность вычислений с плавающей точкой.


## Результаты экспериментов

Ниже приведены результаты измерения времени выполнения умножения матриц на CPU и GPU для различных размеров.

| Размер матрицы | CPU (с) | GPU (с) | Ускорение | Корректность |
|---------------|--------:|--------:|----------:|-------------|
| 100           | 0.3863  | 0.1264  | 3.06×     | True        |
| 480           | 44.9514 | 0.0058  | 7775.79×  | True        |
| 860           | 256.5394| 0.0316  | 8122.16×  | True        |
| 1240          | 793.8334| 0.0941  | 8431.66×  | True        |
| 1620          | 1772.0199| 0.1822 | 9725.72×  | True        |
| 2000          | 3346.4240| 0.3077 | 10877.28× | True        |

### Анализ результатов

- Для небольших матриц (100×100) ускорение невелико из-за накладных расходов и низкой загрузки GPU.  
- При увеличении размера матриц GPU начинает использоваться эффективнее.  
- Время выполнения на CPU растёт значительно быстрее, чем на GPU.  
- Ускорение увеличивается с ростом размера задачи и достигает более **10000×**.  
- Во всех случаях результаты GPU совпадают с CPU, что подтверждает корректность реализации.
