# Matrix Multiplication on CPU and GPU using CUDA

В данной работе реализовано умножение квадратных матриц на CPU и GPU с использованием `Numba CUDA`.  
Программа позволяет сравнить время выполнения последовательной и параллельной реализации, проверить корректность результатов и построить графики зависимости времени выполнения и ускорения от размера матрицы.

---

## Общая идея работы

Умножение матриц — одна из базовых операций линейной алгебры, которая активно используется в численных методах, машинном обучении, компьютерной графике и высокопроизводительных вычислениях.

Для двух матриц `A` и `B` результирующая матрица `C` вычисляется по формуле:

`C[i][j] = Σ A[i][k] * B[k][j]`

Классический алгоритм имеет вычислительную сложность `O(N^3)`, поэтому при увеличении размера матриц время выполнения быстро растет.

Главная особенность задачи состоит в том, что элементы результирующей матрицы можно вычислять независимо друг от друга. Именно поэтому задача хорошо подходит для распараллеливания на GPU.

## Используемые библиотеки

В программе используются следующие библиотеки:

- `math` — для вычисления размеров сетки CUDA
- `time` — для измерения времени выполнения

- `warnings` и `NumbaPerformanceWarning` — для скрытия предупреждений о низкой загрузке GPU при малых размерах матриц

## Описание функций

## `matmul_cpu(a, b)`

Функция выполняет умножение двух матриц на CPU.

### Принцип работы

Реализация основана на классическом алгоритме с тремя вложенными циклами:

- внешний цикл проходит по строкам первой матрицы
- второй цикл проходит по столбцам второй матрицы
- внутренний цикл вычисляет сумму произведений элементов строки на элементы столбца

Каждый элемент результата `c[i][j]` вычисляется последовательно.

### Особенности

- используется стандартный последовательный алгоритм
- вычисления выполняются только на CPU
- распараллеливание отсутствует
- функция служит базовой реализацией для сравнения с GPU-версией

### Проверка размеров

Перед началом вычислений проверяется совместимость размеров матриц:

- число столбцов матрицы `a`
- число строк матрицы `b`

Если размеры несовместимы, выбрасывается исключение `ValueError`.

---

## `matmul_gpu_kernel(a, b, c)`

Это CUDA-ядро, которое выполняется на GPU.

### Назначение

Ядро отвечает за вычисление элементов результирующей матрицы `c`.

### Как работает ядро

Для каждого потока вычисляются координаты:

- `row` — номер строки
- `col` — номер столбца

Эти координаты определяются через:

`cuda.grid(2)`

После этого поток проверяет, попадает ли он в допустимые границы матрицы. Если да, он вычисляет один элемент `c[row][col]`.

### Что именно распараллеливается

Распараллеливание происходит на уровне элементов результирующей матрицы:

- один поток GPU вычисляет один элемент результата
- разные потоки вычисляют разные элементы одновременно

То есть если результирующая матрица имеет размер `N x N`, то для ее вычисления создается множество потоков, и каждый поток обрабатывает свою позицию.

### Что остается последовательным

Несмотря на то, что сама матрица вычисляется параллельно, внутри одного потока остается последовательный цикл по индексу `k`:

- поток берет строку из `a`
- берет столбец из `b`
- последовательно накапливает сумму произведений

Таким образом:

- вычисление разных элементов `c` — параллельное
- вычисление одного конкретного элемента — последовательное

### Граничная проверка

Условие:

`if row < c.shape[0] and col < c.shape[1]`

необходимо для предотвращения выхода за границы массива, поскольку размер сетки CUDA округляется вверх и может содержать лишние потоки.

## `matmul_gpu(a, b, threads_per_block=(16, 16))`

Функция выполняет умножение матриц на GPU и измеряет время выполнения CUDA-ядра.

### Основные этапы работы

1. Проверка совместимости размеров матриц  
2. Копирование входных матриц на устройство  
3. Создание массива результата на GPU  
4. Вычисление размера сетки CUDA  
5. Запуск CUDA-ядра `matmul_gpu_kernel`  
6. Синхронизация устройства  
7. Копирование результата обратно на CPU  

### Выделение памяти на GPU

Входные матрицы передаются на GPU через:

- `cuda.to_device(a)`
- `cuda.to_device(b)`

Результирующая матрица создается на устройстве через:

- `cuda.device_array((n, k), dtype=np.float32)`

### Конфигурация блоков и сетки

В программе используется размер блока:

`threads_per_block = (16, 16)`

Это означает, что в одном блоке находится:

- 16 потоков по одной оси
- 16 потоков по другой оси
- всего 256 потоков на блок

Размер сетки вычисляется динамически:

- `blocks_per_grid_x = ceil(n / 16)`
- `blocks_per_grid_y = ceil(k / 16)`

Это нужно для того, чтобы покрыть всю область результирующей матрицы.

### Измерение времени

Время измеряется только для запуска ядра и выполнения вычислений на GPU:

- запуск ядра
- выполнение
- `cuda.synchronize()`

После синхронизации фиксируется итоговое время `gpu_time`.

Важно отметить, что в данном коде копирование данных на устройство происходит до начала замера времени, а копирование результата с GPU на CPU — после завершения замера.  
Это означает, что измеряемое время GPU относится в первую очередь к вычислительной части ядра, а не ко всей полной операции переноса данных.

## `check_correctness(x, y, eps=1e-3)`

Функция проверяет корректность результатов CPU и GPU.

### Метод проверки

Используется функция:

`np.allclose(x, y, atol=eps, rtol=eps)`

Это позволяет сравнить результаты с учетом небольшой погрешности, возникающей при вычислениях с плавающей точкой.

### Почему это важно

CPU и GPU могут давать незначительно отличающиеся результаты из-за:

- разного порядка вычислений
- особенностей работы с `float32`

Поэтому используется сравнение с допустимой погрешностью, а не проверка на полное побитовое совпадение.

---

## `benchmark_cpu(a, b)`

Функция выполняет запуск CPU-версии и замеряет время ее работы.

### Что делает функция

- запускает `matmul_cpu(a, b)`
- измеряет время выполнения
- возвращает результат умножения и время `cpu_time`

Эта функция используется для удобного сравнения CPU и GPU в общем эксперименте.

---

## `run_experiment(sizes)`

Функция проводит серию экспериментов для разных размеров матриц.

### Что происходит внутри

Для каждого размера `n`:

1. создаются две случайные квадратные матрицы `n x n`
2. выполняется умножение на CPU
3. выполняется умножение на GPU
4. проверяется корректность результата
5. вычисляется ускорение:
   
   `speedup = cpu_time / gpu_time`

6. результаты добавляются в список словарей
7. информация выводится в консоль в виде таблицы

### Формат сохраняемых результатов

Для каждого размера сохраняются:

- `size`
- `cpu_time`
- `gpu_time`
- `speedup`
- `correct`

Это удобно для последующего построения графиков.

## `plot_results(results)`

Функция строит два графика по результатам экспериментов.

### Первый график

**Matrix multiplication time: CPU vs GPU**

Показывает зависимость времени выполнения от размера матрицы для двух реализаций:

- CPU
- GPU

### Второй график

**GPU speedup**

Показывает, как изменяется ускорение GPU относительно CPU в зависимости от размера матрицы.


### Назначение графиков

Графики позволяют наглядно увидеть:

- насколько быстро растет время CPU
- насколько медленнее растет время GPU
- как увеличивается ускорение при росте размера входных данных


Ключевая часть всей лабораторной — именно распараллеливание вычислений на GPU.

### На CPU

В функции `matmul_cpu` вычисления идут строго последовательно:

- сначала один элемент
- потом следующий
- потом следующий

Даже если алгоритм логически работает с матрицей, фактически операции выполняются по одной.

### На GPU

В `matmul_gpu_kernel` распараллеливание происходит следующим образом:

- создается двумерная сетка потоков
- каждый поток получает координаты `row` и `col`
- поток вычисляет только один элемент результирующей матрицы

То есть параллелизуется не внутренний цикл умножения, а множество независимых вычислений элементов результата.

### Почему это эффективно

Потому что элементы матрицы результата не зависят друг от друга.  
Чтобы вычислить `c[5][7]`, не нужно знать `c[5][8]` или `c[6][7]`.

Именно это делает задачу удобной для GPU: можно одновременно запустить тысячи потоков, каждый из которых будет занят своим элементом.

---

## Почему GPU показывает ускорение

GPU эффективен в этой задаче по следующим причинам:

### 1. Большое количество потоков

GPU может одновременно выполнять огромное количество потоков, в то время как CPU выполняет операции последовательно или ограниченным числом ядер.

### 2. Независимость элементов результата

Так как элементы матрицы `c` независимы, задача очень хорошо распараллеливается.

### 3. Массовая однотипная нагрузка

Все потоки выполняют одну и ту же операцию:

- берут строку
- берут столбец
- считают сумму произведений

Такая нагрузка хорошо подходит архитектуре GPU.

---

## Особенности получаемых результатов

### Малые размеры матриц

Для небольших матриц GPU может использоваться не полностью.  
Это связано с тем, что число блоков и потоков еще недостаточно велико, чтобы хорошо загрузить устройство.

### Большие размеры матриц

С ростом размера матриц:

- увеличивается количество потоков
- GPU загружается лучше
- ускорение становится более выраженным

### Важное замечание по времени GPU

В твоем коде время GPU измеряется вокруг запуска ядра и синхронизации.  
Передача входных матриц на GPU выполняется до начала измерения, а копирование результата обратно — после окончания измерения.

Поэтому сравнение отражает прежде всего скорость вычисления ядра, а не полную стоимость всей GPU-операции вместе с обменом данными.

Это нормально, если цель лабораторной — показать ускорение параллельного вычисления, но этот момент полезно явно отметить в отчете.
